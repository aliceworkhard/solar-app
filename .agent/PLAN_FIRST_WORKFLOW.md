# Plan First Workflow

更新时间：2026-04-27

## 目的

后续涉及修改、开发、修复、协议调整、UI 调整、构建发布、文档结构调整时，先输出方案并更新 `todo.md`，等用户看过方案确认后再实施。

## 适用范围

必须先走方案：

- 改业务代码。
- 改 BLE 通信。
- 改协议命令、解析、校验。
- 改 UI 页面、交互、样式。
- 改 Android 原生插件、权限、构建配置。
- 改任务结构、协作规则、发布流程。

可以直接做：

- 只读分析。
- 简短问答。
- 用户明确要求的很小文案修正。
- 本流程自身的 TODO / plan / approval 记录维护。

## 执行步骤

1. 读取 `AGENTS.md`、`.agent/START_HERE.md`、`.agent/AI_CONTEXT.md`、`todo.md`。
2. 读取与任务直接相关的文件。
3. 在 `.agent/plans/` 新建计划文件。
4. 在 `todo.md` 写入任务、优先级、状态、验收标准。
5. 回复用户：方案摘要 + 等待确认。
6. 用户确认后，在 `.agent/approvals/` 写审批记录。
7. 只修改批准范围内的文件。
8. 运行验证。
9. 更新 `todo.md`、`.agent/logs/`、必要时更新 `.agent/CHANGE_INDEX.md`。

## 确认口径

以下回复视为批准：

- `同意`
- `批准执行`
- `按方案做`
- `开始修改`
- `执行`
- `可以，做吧`

如果用户只是继续讨论、补充资料或提出问题，不视为批准。

## 输出要求

方案必须包含：

- 当前事实。
- 目标。
- 修改范围。
- 不做范围。
- 涉及文件。
- TODO 更新。
- 风险。
- 验证计划。
- 回滚方案。
- 等待确认。
