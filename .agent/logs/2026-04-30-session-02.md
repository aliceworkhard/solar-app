# Session Log - 2026-04-30 - T034 Time Control Mode Write

## Summary

Implemented the user-approved T034 plan for protocol time-control mode.

Changes:

- Added typed time-control parameter model, validation, encoder, decoder, and draft-change helper.
- Added `CommandBuilder.writeTimeControlParams()` for `B1 MODE=01` full-frame writes.
- Fixed `responseParser` mode mapping so `MODE=01` is time mode and `MODE=02` is radar mode.
- Added `DeviceStatus.timeControlParams` and `DeviceController.writeTimeControlParams()`.
- Added a compact time-control editor to the control panel:
  - steppers for battery type, battery voltage, light delay, and sensitivity;
  - sliders for max PWM, active segment duration/power, morning duration/power;
  - five segment buttons and a summary strip.
- The App syncs the editor draft from `status.timeControlParams` after read-params notify parsing.
- Sync is guarded by the `timeControlParams` object reference, so normal E1 status refreshes do not revert a locally edited draft to an older B1 snapshot.

Preserved:

- Existing 5 MVP commands remain unchanged.
- Ordinary commands and time-control write do not wait for BLE responses by default.
- UI does not parse HEX.
- Source protocol Excel and old APK files were not touched.
