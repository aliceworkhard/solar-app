# Session Log - 2026-04-29 - T-027 Android Edge-To-Edge Insets

## Goal

Fix Android system bar edge-to-edge behavior without relying on only status/navigation bar colors or root-level hardcoded padding.

## Approval

- Plan: `.agent/plans/2026-04-29-t027-android-edge-to-edge-insets.md`
- Approval: `.agent/approvals/2026-04-29-t027-android-edge-to-edge-insets.md`
- User confirmation: “按方案做”

## Root Cause

- `MainActivity` still mixed manual `setStatusBarColor` / `setNavigationBarColor` calls with old `SYSTEM_UI_FLAG_LAYOUT_FULLSCREEN` and `SYSTEM_UI_FLAG_LAYOUT_HIDE_NAVIGATION` behavior.
- Real Android system insets were not bridged from the native `WindowInsetsCompat` layer into the WebView DOM.
- CSS used root `.shell` top/bottom safe padding, which moved the whole app instead of letting the background draw behind system bars while only moving content.

## Changes

- `MainActivity.java`
  - Added `WindowCompat.enableEdgeToEdge(window)`.
  - Removed old `SYSTEM_UI_FLAG_*` fullscreen/navigation flags and manual system-bar color setup from the Activity path.
  - Set light status/navigation bar icons with `WindowInsetsControllerCompat`.
  - Disabled Android Q+ status/navigation bar contrast enforcement.
  - Added cutout layout handling.
  - Added `ViewCompat.setOnApplyWindowInsetsListener` on the WebView and returned the original insets, not `CONSUMED`.
  - Injected `--system-top/right/bottom/left` CSS variables into the WebView.
- `styles.xml`
  - Kept transparent system bars.
  - Added contrast enforcement and cutout theme safeguards.
- `app/build.gradle`
  - Added explicit `androidx.core:core`.
- `src/app.ts`
  - Added `SystemInsets` helpers and `window.solarRemoteApplySystemInsets`.
- `src/styles.css`
  - Removed root `.shell` top/bottom inset padding.
  - Applied top inset only to `.app-header`.
  - Applied bottom inset only to `.bottom-nav`.
  - Applied bottom content spacing to `main`.
- `src/app.test.ts`
  - Added a regression test for mapping Android insets to CSS variables.
- `.agent/reports/2026-04-29-t027-edge-to-edge-layout-check.cjs`
  - Added local Chrome layout validation with simulated top/bottom insets.
