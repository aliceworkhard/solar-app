## Changes

- Confirmed Capacitor package versions remain `@capacitor/core@8.3.1`, `@capacitor/android@8.3.1`, `@capacitor/cli@8.3.1`; `SystemBars` config stays enabled.
- `MainActivity.java`
  - Kept early `WindowCompat.enableEdgeToEdge(window)` and added immediate decor insets request.
  - Removed the WebView no-op `WindowInsets` listener so it does not compete with Capacitor SystemBars parent listener.
  - Kept the main listener on DecorView and returned the original insets, not `WindowInsetsCompat.CONSUMED`.
  - Split full edge insets from horizontal content-safe insets: side gesture/tappable values are still available to Web as native edge values, but only `systemBars/displayCutout` left/right become content-safe padding.
  - Moved native top/bottom paint fallback views into the content frame at the back layer; views remain `clickable=false`, `focusable=false`, `importantForAccessibility=no`, and only paint the top/bottom inset heights.
  - Injected raw px values plus `density`, `webViewWidthPx`, and diagnostic flag into Web instead of relying only on density-converted strings.
  - Added `EDGE_DIAGNOSTIC_COLORS = false`; when manually enabled in a debuggable build, decor is magenta, content is green, and Web DOM can show lime. Formal APK has it off.
- `src/app.ts`
  - Added content-safe CSS variables and native inset metadata.
  - Added `resolveNativeCssPx()` width-ratio fallback: if `window.innerWidth` does not match `webView.getWidth()/density`, CSS px use `window.innerWidth / webViewWidthPx`.
  - Added runtime metadata at `window.__nativeSystemInsetsMeta` and logs one info message if ratio fallback is used.
- `src/styles.css`
  - Changed `--edge-*` to use `max(native, system, safe-area, env)` and kept bottom nav `bottom: 0`.
  - Added `--content-safe-left/right`.
  - Restored `.shell` base horizontal padding to 14/12/10px plus only explicit content-safe insets, so ordinary side gesture insets no longer compress the UI.
  - Bottom nav left/right padding now uses content-safe insets; bottom padding still uses `--edge-bottom`.
  - Added transparent-by-default Web diagnostic overlay variable.
- `src/app.test.ts`
  - Added failing-first tests for content-safe side inset separation and native px width-ratio fallback.
