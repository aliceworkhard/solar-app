# 第一阶段执行任务清单（通信打通，3-5 天）

更新时间：2026-04-22

## 1. 阶段目标

- 在安卓真机完成 BLE 基础闭环：
  - 扫描设备
  - 连接设备
  - 发现服务和特征
  - `FFF1` 写入
  - `FFF2` 通知接收
  - 日志记录
- 产出：可运行调试版 APK + 参数确认结论（含 `writeType`）。

## 2. 按天任务拆解

## Day 1：环境与骨架

1. 初始化安卓工程与基础页面（扫描页/调试页）。
2. 接入 BLE 权限申请与蓝牙开关状态检测。
3. 实现设备扫描与列表展示（名称、RSSI、地址/标识）。
4. 支持按名称前缀 `AC632N` 过滤。

完成标准：
- 能稳定扫描到目标设备。
- 权限拒绝/蓝牙关闭有明确提示。

## Day 2：连接与服务发现

1. 实现连接、断开、重连基础流程。
2. 完成服务发现，确认 `FFF0` 存在。
3. 列出 `FFF0` 下特征值，确认 `FFF1`、`FFF2`。
4. 订阅 `FFF2 notify` 并展示订阅状态。

完成标准：
- 可重复连接成功。
- 可稳定进入已订阅状态。

## Day 3：写入与回包

1. 在调试页支持 HEX 输入与发送。
2. 对 `FFF1` 做写入，支持 `WRITE` 与 `WRITE_NO_RESPONSE` 切换。
3. 在 `FFF2` 接收回包并显示时间戳/HEX。
4. 记录请求和回包对应关系（最少命令 ID + 时间）。

完成标准：
- 至少 3 条命令可稳定收发。
- `writeType` 得到明确结论。

## Day 4：稳定性与异常（可并入 Day 5）

1. 处理连接中断自动回收状态。
2. 增加发送超时与失败提示。
3. 处理无回包、校验失败、空包等异常日志。
4. 增加基础重试策略（仅调试命令）。

完成标准：
- 连续测试 30 分钟无崩溃。
- 断开后可恢复到可重连状态。

## Day 5：验收与归档（可选）

1. 按验收清单逐条验证并记录。
2. 固化《通信参数确认表》最终值。
3. 固化《最小命令集表》至少 5 条命令。
4. 输出阶段结论和下一阶段输入。

完成标准：
- 阶段验收通过。
- 文档可直接交给后续协议/UI开发使用。

## 3. 阶段验收清单

1. 能扫描并连接目标设备。
2. 能发现 `FFF0 / FFF1 / FFF2`。
3. 能稳定启用 `FFF2 notify`。
4. 能向 `FFF1` 写入并收到回包。
5. 已确认 `writeType`。
6. 日志可查看（至少含时间、方向、HEX、结果）。

## 4. 阻塞项与处理

| 阻塞项 | 识别方式 | 处理建议 |
| --- | --- | --- |
| 扫描不到设备 | 多机型均搜不到 | 先确认设备广播状态和供电 |
| 连接后无 `FFF0` | 服务列表无目标服务 | 核对设备型号与 profile 匹配 |
| 可写不可收 | `FFF1` 写成功但无 notify | 检查 `FFF2` 订阅是否成功 |
| 回包断续 | 丢包或半包 | 开启缓冲区拼包与节流发送 |
| 写入失败 | 特征属性不匹配 | 切换 `WRITE/WRITE_NO_RESPONSE` |

## 5. 阶段输出物

1. 调试版 APK（内部）。
2. 《通信参数确认表》更新版。
3. 《最小命令集表》更新版。
4. 调试日志样例（不少于 20 条收发记录）。
